# 변경사항

## 2026-07-14

### 1. 간투사 일괄 선택 설정 기능 추가 (컷 편집)

**배경**
- 기존 "간투사 선택" 버튼은 화자·텍스트 구분 없이 **모든 간투사를 일괄 선택**해, 사용자가 원하지 않는 간투사까지 전부 선택되는 불편이 있었습니다.

**변경 내용**
- "간투사 선택" 버튼 오른쪽 끝에 **설정(⚙) 버튼**을 버튼 내부에 추가. 클릭 시 설정 다이얼로그가 열립니다.
- 설정 다이얼로그 구성:
  - **선택할 간투사**: 간투사 텍스트를 **중복 제거**해 목록으로 표시. 체크된 텍스트만 일괄 선택 대상이 됩니다.
  - **선택할 화자**: 간투사가 있는 화자가 **2명 이상일 때만** 표시. 체크된 화자의 간투사만 일괄 선택됩니다.
  - 각 섹션에 **전체 선택 / 전체 해제** 토글.
  - 상단에 현재 조건으로 **선택될 간투사 개수**를 실시간 표시.
- 필터는 **세션 동안 유지**되고, 새 받아쓰기(화면 초기화) 시 초기화됩니다.
- 기본값은 전부 선택 → **기존 동작과 동일**.

**개수 집계 규칙 (비대칭)**
- **텍스트 개수** = 현재 선택된 **화자 기준**. 화자를 껐다 켜면 개수와 총계가 실시간으로 바뀝니다.
- **화자 개수** = 그 화자의 **고유 간투사 총량** (텍스트 필터 무관). 화자가 통째로 "선택할 게 없음"으로 보이는 문제를 방지합니다.

**항목 상태**
- **정상**: 개수 표시, 선택 가능.
- **0 (회색)**: 남아있지만 전부 미선택 화자 소속 → 지금은 선택 불가(비활성).
- **삭제됨**: 컷 적용으로 전부 삭제됨. 목록에는 유지하되 선택 불가(비활성) 처리 — 삭제 후 항목이 사라져 보이던 문제 해결.

**관련 파일**
- `client/src/components/FillerSettingsDialog.jsx` (신규)
- `client/src/hooks/useWordSelection.js` (필터 상태·개수 집계 로직)
- `client/src/components/CutEditTab.jsx` (설정 버튼·다이얼로그 연결)
- `client/src/components/App.jsx` (props 배선)

### 2. 배경 탭 — 부별 추출 버튼 개선

**배경**
- 부별 요소 추출 버튼이 항상 새로고침 아이콘이라, 처음 추출 전인데도 "다시 추출"처럼 보여 혼동을 주었습니다.

**변경 내용**
- 부별 버튼을 요소 유무에 따라 분기:
  - **요소 없음(초기)**: `Sparkles` 아이콘 + **"추출"** 라벨 (전체 추출 버튼과 동일 아이콘).
  - **요소 있음**: 새로고침(`RefreshCw`) 아이콘 = 다시 추출.
  - **추출 중**: 스피너.
- 툴팁도 상태별로: "이 부 요소 추출" / "이 부만 다시 추출".
- 빈 상태 안내 문구에 오른쪽 "추출" 버튼 언급 추가.

**관련 파일**
- `client/src/components/BackgroundTab.jsx`

### 3. 컷 편집 툴바 UI 다듬기

- 무음 선택 · 간투사 선택 · 설정(⚙) 버튼의 **호버 하이라이트 톤 통일** (`neutral-700`).
- "간투사 선택" 버튼과 내부 설정 버튼을 **독립적인 두 영역**으로 분리 — 각각 호버한 쪽만 하이라이트됩니다.
- 설정 버튼 클릭 영역을 버튼 오른쪽 끝까지 확장하고 포인터 커서·구분선 적용.
